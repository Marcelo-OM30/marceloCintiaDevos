## Problemas Identificados e Resolvidos

### 1. App travava ao abrir (crash na inicialização)

**Causa principal:** `expo-constants` estava ausente nas dependências. O `expo-router` exige essa biblioteca como peer dependency obrigatória. Sem ela, o app crasha ao tentar inicializar o roteamento.

**Solução:**
```bash
npx expo install expo-constants -- --legacy-peer-deps
```

---

### 2. Versões de pacotes fora do esperado pelo Expo SDK 54

O `expo doctor` apontou 4 pacotes com versão incorreta:

| Pacote | Esperado | Encontrado |
|---|---|---|
| `react` | 19.1.0 | 19.2.6 |
| `react-dom` | 19.1.0 | 19.2.6 |
| `@types/react` | ~19.1.10 | 19.2.14 |
| `react-native-worklets` | 0.5.1 | 0.5.2 |

**Solução:**
```bash
npx expo install react@19.1.0 react-dom@19.1.0 @types/react@19.1.10 react-native-worklets@0.5.1 -- --legacy-peer-deps
```

Para `@types/react` (tipagem TypeScript, não afeta o APK), foi adicionada exclusão da validação no `package.json` para evitar warning persistente no build:

```json
"expo": {
  "install": {
    "exclude": ["@types/react"]
  }
}
```

---

### 3. Ícone do app — Ictus (peixe cristão)

O ícone foi alterado de placeholder Expo para um peixe ictus ciano (`#00C8D7`) em fundo preto.

**Script de geração:** `scripts/generate-icons.mjs`

- `assets/icon.png` (1024×1024)
- `assets/adaptive-icon.png` (1024×1024)
- `assets/splash-icon.png` (512×512)
